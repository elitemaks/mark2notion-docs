### What to Include in Bug Reports

To help us resolve issues quickly, please include:

<AccordionGroup>
  <Accordion title="Request Details">
    - HTTP method and endpoint
    - Request headers (without API key)
    - Request body (sample data)
    - Expected vs actual behavior
  </Accordion>
  
  <Accordion title="Error Information">
    - Full error response
    - HTTP status code
    - Timestamp when the error occurred
    - Frequency (one-time or recurring)
  </Accordion>
  
  <Accordion title="Environment Details">
    - Programming language/framework
    - SDK version (if applicable)
    - Operating system
    - Any relevant code snippets
  </Accordion>
</AccordionGroup>

### Example Bug Report

**Describe the bug**  
The append endpoint returns a 500 error when appending large markdown content (>5000 characters).

**Request**
```http
POST /api/append
Headers: x-api-key: [REDACTED]
Body: {
  "markdown": "[large content here]",
  "notionToken": "[REDACTED]", 
  "pageId": "abc123..."
}
```

**Error Response**
```json
{
  "status": "error",
  "message": "Internal server error"
}
```

**Expected Behavior**  
Content should be successfully appended to the Notion page.

**Environment**
- Language: JavaScript/Node.js
- OS: macOS 14.1
- Content size: ~6000 characters

### Popular Feature Requests

Some features that are commonly requested:

- **Bulk operations**: Process multiple pages in one request
- **Webhook support**: Get notified when operations complete
- **Template support**: Use predefined page templates
- **Advanced formatting**: Support for more Notion block types
- **Rate limit optimization**: Higher limits for enterprise users
